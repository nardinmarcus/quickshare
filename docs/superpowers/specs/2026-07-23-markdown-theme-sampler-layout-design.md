# QuickShare Markdown 主题样例布局设计

- 日期：2026-07-23
- 状态：已确认设计；尚未授权生产实现
- 范围：首页 Markdown 发布工作台的 Theme Sampler 布局、内容密度、响应式与失败边界；管理页只共享精简样例内容
- 依据：QuickShare Sharing Context、ADR-0001、2026-07-22 Markdown Theme Catalog 正式规格、四方向视觉比较与逐项设计质询

## 1. 问题

首页在识别为 Markdown 内容后，会在内容输入与发布设置之间展示 Theme Sampler。当前 Sampler iframe 在桌面固定为 440 px 高、移动端固定为 650 px 高，并作为一个完整区块进入主表单的纵向文档流。这使辅助性的主题比较内容占据页面主要面积，把标题、描述、到期日期、访问方式和发布操作推到首屏以下。

问题不只是预览尺寸偏大，而是信息层级错位：Theme Sampler 被呈现得像一份完整文档，而它的实际职责只是使用固定内容帮助创建者快速比较 Markdown Theme Preset。用户真实正文已经由独立的“安全预览”负责，不应在 Sampler 中重复承担。

## 2. 设计结论

采用“右侧检查器布局 + 紧凑主题标本内容”的组合方案：

- 桌面端以右侧窄栏持续展示主题选择和固定样例。
- Sampler 从完整测试文档精简为五类高识别度元素。
- 编辑、发布设置与操作按钮保留在左侧主流程，并在默认桌面视口完整进入首屏。
- 小屏改为上下布局，但仍默认显示紧凑样例，不引入抽屉、弹窗或二次打开。

该方案使用右侧检查器作为空间结构，但“右侧检查器”不是新的领域概念。领域与界面规范名称保持为 Theme Sampler／“主题样例”。

## 3. 目标

1. 恢复首页发布工作台的主次关系，让编辑和发布设置成为首屏主体。
2. 保留切换主题后立即比较视觉风格的能力。
3. 明确区分固定“主题样例”和真实内容“安全预览”。
4. 在桌面、小屏和管理页之间维持一致的样例语义。
5. 不改变 Catalog、主题持久化、安全预览、Share 创建或管理编辑的业务契约。

## 4. 非目标

- 不在 Theme Sampler 中渲染用户当前输入的 Markdown。
- 不合并或替换现有“安全预览”。
- 不新增主题画廊、主题卡片墙、主题编辑器或自定义 CSS。
- 不重新设计管理页布局。
- 不修改 Creator Theme Preference、Catalog 回退或 Share 存储语义。
- 不新增数据库、API、认证、权限或发布状态。
- 不添加吸底操作栏、主题切换动画、骨架屏或远程加载状态。
- 本规格不授权修改生产代码、提交、推送或部署。

## 5. 领域与界面语言

- **Theme Sampler／主题样例**：使用固定代表性 Markdown 内容比较当前 Preset，不展示用户正文。
- **安全预览**：展示用户真实待发布内容，继续使用现有按需预览流程。
- 界面不得使用“主题预览”或“预览面板”称呼 Theme Sampler，以免与安全预览混淆。
- “右侧检查器”只描述桌面布局形态，不进入领域词汇或持久化模型。

## 6. 首页桌面布局

当视口宽度达到 1024 px 时，首页发布卡片使用左右双栏：

```text
┌──────────────────────────────────────────────────────────┐
│ 内容输入／发布设置／访问方式／操作      │ Markdown 主题    │
│                                          │ 固定主题样例     │
│                                          │                 │
└──────────────────────────────────────────────────────────┘
```

- 整体保持一个统一工作台卡片，不拆成两个独立悬浮卡片。
- 左栏使用剩余可用宽度，最小宽度可以收缩但不得产生页面级横向溢出。
- 右栏目标宽度为 320 px，最大不超过 340 px，不按固定百分比随宽屏继续放大。
- 左右区域使用间距和一条弱分隔线建立层级，不叠加第二套重边框和阴影。
- Markdown 主题下拉框位于右栏顶部，与主题样例构成一个完整单元。
- 右栏可以有限吸顶，使主题选择和样例在左侧表单滚动期间保持可见。
- 右栏及其 iframe 不得产生独立纵向滚动条；内容必须在紧凑高度内完整呈现。

## 7. 小屏与中等宽度布局

当视口宽度低于 1024 px 时：

- 页面统一改为上下布局，不保留压缩后的桌面双栏。
- 主题下拉框和主题样例保持为一个完整单元。
- 主题样例默认可见，目标高度为 180–220 px。
- 不折叠到“查看主题样例”按钮，不使用抽屉或模态弹窗。
- 页面和 Sampler 均不得产生水平溢出或嵌套纵向滚动。
- 不要求完整发布流程进入移动端首屏，但内容顺序、焦点顺序和操作顺序必须一致。

## 8. 主题样例内容

首页和管理页共享同一份精简固定样例，只保留五类最能区分 Preset Visual Identity 的元素：

1. 标题
2. 正文与链接
3. 引用
4. 围栏代码块
5. 表格

任务列表、键盘键、图片、流程图和其他阅读基线测试元素不再进入交互式 Theme Sampler。它们继续由主题自动化测试、浏览器矩阵和实际安全预览覆盖。

本节细化并取代 2026-07-22 Markdown Theme Catalog 正式规格中关于 Sampler 必须展示完整代表元素集合的要求；Catalog、Theme Reading Baseline 和公开 Share 的完整内容支持不变。

## 9. 视觉层级

- 发布工作台只使用一个主要卡片表面。
- Theme Sampler 是辅助区域，其背景、分隔和文字层级应低于编辑与发布主流程。
- 发布按钮继续是页面唯一的实色主操作。
- “安全预览”保持次级操作样式，不与发布按钮竞争。
- “发布设置”始终完整展开，不改为折叠面板或“高级设置”。
- 操作区保持普通文档流，不吸底、不悬浮、不遮挡表单。

## 10. 交互与状态

- 主题切换后立即更新固定样例。
- 不增加淡入淡出、骨架屏、加载动画或人为延迟。
- 保留控件自身现有的 hover、active 与 focus-visible 反馈。
- 首页继续读取和更新 Creator Theme Preference；管理页仍以 Share 已存主题为准，并且不读写创建偏好。
- 切换主题不得触发安全预览，也不得修改标题、正文、访问方式或其他发布字段。
- 用户请求安全预览或发布时，继续使用当时选择的合法 Catalog ID。

## 11. 失败边界

Theme Sampler 是辅助比较工具，不是发布前置条件：

- Sampler 初始化或渲染失败时，在样例区域显示安静的“样例暂不可用”提示。
- 失败不得禁用主题下拉框、安全预览或发布按钮。
- 失败不得自动改写或清空用户已经选择的主题。
- Catalog 解析、请求校验和公开渲染继续使用现有服务端回退契约。
- Sampler 故障不能被描述为 Share 内容预览失败。

## 12. 首页与管理页边界

- 右侧检查器布局只应用于首页发布工作台。
- 管理页保留现有主题控件位置和整体表单结构。
- 首页与管理页共享同一 Theme Sampler partial、同一固定样例内容和同一浏览器控制器。
- 上下文差异只体现在外部布局和尺寸，不允许派生两套样例结构或主题资源逻辑。

## 13. 安全与无障碍

- Theme Sampler 继续使用无脚本权限的 sandbox iframe 和受信任的服务端 Catalog 资源投影。
- 浏览器不得从主题 ID、localStorage 或用户输入拼接样式资源地址。
- 下拉框必须有可见标签，并通过可访问关系指向主题样例。
- 样例区域必须有随当前 Preset Label 更新的可访问名称。
- iframe 不进入普通 Tab 顺序；键盘用户通过主题下拉框完成选择。
- 所有布局在 200% 缩放下必须保持可操作且无页面级横向溢出。
- `prefers-reduced-motion` 用户不会因为本功能新增内容动画。

## 14. 首屏验收标准

在 1280 × 800 及以上桌面视口的默认状态下，无需页面滚动即可看到：

- 内容输入区
- Markdown 主题选择和主题样例
- 标题、描述与到期日期
- 访问方式
- 安全预览与发布按钮

1440 × 900 是舒适展示状态，不是唯一可通过的截图尺寸。展开自定义密码、错误详情、发布回执或安全预览后的动态状态不受默认首屏高度约束。

## 15. 验证矩阵

实现后至少验证以下组合：

| 视口 | 预期布局 | 关键检查 |
| --- | --- | --- |
| 1440 × 900 | 双栏 | 完整工作台舒展、右栏不膨胀、全部默认操作可见 |
| 1280 × 800 | 双栏 | 完整默认发布流程进入首屏 |
| 1024 × 768 | 双栏边界 | 左栏可用、右栏约 320 px、无横向溢出 |
| 1023 × 768 | 上下布局边界 | 无断点抖动、样例紧凑可见 |
| 768 × 1024 | 上下布局 | 字段与样例顺序正确、无嵌套滚动 |
| 375 × 812 | 单列 | 样例 180–220 px、控件可读、无页面级横向溢出 |

浏览器验收还必须覆盖：

- Markdown 与非 Markdown 内容切换时 Theme Sampler 正确显示和隐藏。
- 十二个 Preset 切换后样例立即更新且保持五类元素可辨识。
- Theme Sampler 故障不会阻塞安全预览和发布。
- 首页偏好记忆与管理页已存主题边界不变。
- 键盘操作、可见焦点、可访问名称和 200% 缩放通过。
- 全量测试不回归 HTML、SVG、Mermaid、访问方式、安全预览和 Share 发布。

## 16. 取舍与风险

- 固定右栏牺牲一部分桌面编辑宽度，换取主题选择的持续可见；320–340 px 上限用于保护编辑区。
- 默认可见的小屏样例仍占用一定纵向空间，换取选择主题后的即时视觉反馈。
- 精简样例不再证明每一种 Markdown 元素的呈现质量，因此完整元素覆盖必须留在自动化测试和安全预览中。
- 首页与管理页共享内容但不共享布局，需要通过清晰的上下文样式边界避免首页规则泄漏到管理页。
