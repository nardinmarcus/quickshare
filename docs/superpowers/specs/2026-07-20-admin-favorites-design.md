# QuickShare 管理后台收藏设计

- 日期：2026-07-20
- 状态：Issues #6–#8 已实施并进入 `main`；Issue #9 正在完成全量验收与安全发布
- 范围：管理后台分享列表、分享详情、筛选、导出与审计；不改变公开分享行为

## 1. 目标

在管理后台为值得保留和再次查看的分享增加全局收藏状态。管理员可以从分享列表或详情页收藏、取消收藏，并通过“已收藏”筛选快速找到这些内容。

收藏状态必须以 PostgreSQL 持久数据为生产环境唯一真值。浏览器状态、Cookie、运行时内存与缓存都不能成为收藏状态的权威来源；Memory Repository 只用于本地开发和测试语义对齐。

## 2. 已确认的产品行为

1. 收藏是管理后台全局状态，不区分管理员个人身份；所有后台会话看到相同结果。
2. 收藏是二元分类，只区分收藏与未收藏，不记录收藏时间，也不提供收藏专属排序。
3. 筛选只提供“全部”和“已收藏”，并可与搜索、内容类型、保护状态和日期条件组合。
4. 收藏按钮同时出现在分享列表每行和分享详情页顶部；公开分享页不显示该能力。
5. 克隆已收藏分享时，新分享默认为未收藏，原分享保持不变。
6. 后台 JSON 导出增加 `isFavorite`。
7. 每次真实状态变化记录 `page.favorite.update` 审计事件，详情包含 `from` 与 `to`；幂等提交不写审计。
8. 审计写入失败不回滚已经成功的收藏变化，但必须记录服务端错误。
9. 收藏与公开过期互相独立：过期分享仍可收藏、取消收藏和筛选；删除分享时收藏状态随实体一起消失。
10. 在“已收藏”结果中取消收藏后，等待服务端成功，再刷新当前筛选结果，使该行消失并重新计算总数和分页。
11. 第一版不提供批量收藏或批量取消收藏。
12. 未收藏使用中性色空心星；已收藏使用现有青色强调的实心星，同时提供文字、`aria-pressed` 和明确的可访问名称。

## 3. 非目标

本次不包含：

- 管理员账号、个人收藏、团队归属或权限系统。
- 收藏时间线、最近收藏排序、收藏备注、文件夹、标签或多级分类。
- 批量收藏、批量取消收藏或导入收藏状态。
- 在首页、公开分享页、近期分享接口或 Share API 中暴露收藏状态。
- 收藏数量统计、后台统计卡片或公开精选列表。
- 用 Local Storage、Cookie、运行时缓存、Redis/KV 或分析系统保存收藏真值。
- React/Vue 重写、ORM 引入或 `app.js` 的无关重构。

## 4. 领域与数据模型

领域词汇以根目录 `CONTEXT.md` 为准：收藏对象是 **Share**，收藏后的实体称为 **Favorite Share**，不称为“收藏链接”或“个人收藏”。

新增 additive migration `003_page_favorites.sql`：

```sql
ALTER TABLE public.pages
  ADD COLUMN IF NOT EXISTS is_favorite BOOLEAN NOT NULL DEFAULT FALSE;
```

约束：

- 所有历史分享迁移后均为未收藏，不需要回填脚本。
- 新建分享依赖数据库默认值 `FALSE`；Memory Repository 显式初始化为 `false`。
- 不新增 `favorited_at`。
- 第一版不新增收藏索引。当前数据量与单一布尔筛选不需要提前优化；只有真实查询测量证明必要时再增加索引。
- 不修改已经部署的 `001_baseline.sql` 或 `002_site_settings.sql`；空库按 `001 -> 002 -> 003` 顺序构建。

## 5. Repository 边界

Postgres 与 Memory Repository 增加一致的收藏行为：

- `setFavorite(id, isFavorite)`：严格接收布尔值，返回 `{ found, changed, isFavorite, previousValue }`。
- 目标不存在时返回 `found=false`。
- 目标值与当前值相同时返回 `changed=false`，不执行额外状态写入。
- 真实变化时返回变化前后的布尔值，供路由生成审计事件。

现有读取行为同步扩展：

- `getById()` 返回 `is_favorite`，供详情页使用。
- `listAdminPages()` 返回 `is_favorite`，并支持 `isFavorite=true` 条件。
- `countPages()` 使用与列表完全相同的收藏条件，确保总数和分页一致。
- Postgres 与 Memory 的搜索、类型、保护状态、日期和收藏条件必须可组合。
- `listRecent()`、公开读取、浏览量统计和后台统计保持现状，不投影收藏状态。

生产查询只使用数据库列，不添加进程级收藏缓存。列表继续沿用现有创建时间排序；收藏不改变顺序。

## 6. 收藏接口

新增幂等管理接口：

```http
PUT /admin/pages/:id/favorite
Content-Type: application/json
X-CSRF-Token: <dashboard csrf token>

{"isFavorite": true}
```

边界：

- 使用 JSON 语义的 Dashboard 鉴权，缺失或过期会话返回 `401` JSON，不重定向到 HTML 登录页。
- 必须通过现有 Dashboard CSRF 校验；失败返回 `403` JSON。
- `isFavorite` 必须是布尔值；缺失、字符串或数字均返回 `400`，数据库不变。
- 分享不存在返回 `404`。
- 成功返回 `200`：`{ success, changed, isFavorite }`。
- Repository 写入失败返回 `500`，前端保持原状态。

真实变化成功后调用现有审计 Repository：

```json
{
  "action": "page.favorite.update",
  "pageId": "<share id>",
  "details": { "from": false, "to": true }
}
```

审计沿用普通页面更新的尽力记录边界：收藏写入成功即视为操作成功；审计失败只写服务端错误日志，不向响应中泄露连接信息、会话、CSRF token 或分享正文。

## 7. 列表筛选与 URL 语义

后台列表使用 `favorite=true` 表示“只看收藏”；缺失该参数表示“全部”。不增加 `favorite=false`，因为第一版没有“未收藏”筛选。

`GET /admin/pages` 将该值规范化为 `filterOptions.isFavorite`，同时传给 `countPages()` 与 `listAdminPages()`。模板获得 `filters.favorite` 用于选中状态和 URL 生成。

列表中的搜索、日期、类型、状态、排序、分页和清除链接必须保留其他仍生效的条件。实现可增加一个小型服务端 URL 构造辅助函数，集中删除空值和覆盖单个条件，避免继续手写容易遗漏参数的查询字符串；不借此重构其他路由。

空状态必须区分：

- 系统确实没有任何分享：显示“还没有分享”。
- 当前组合条件没有结果：显示“没有符合条件的分享”，并提供清除筛选入口。

## 8. 后台交互

### 8.1 列表页

- 在每行操作区增加星标按钮，与克隆和删除保持同一操作层级。
- 按钮携带分享 ID、当前服务端状态和是否处于收藏筛选结果。
- 未收藏：中性色空心星，文案与可访问名称为“收藏分享 {标题}”。
- 已收藏：实心青色星，文案与可访问名称为“取消收藏 {标题}”。
- 使用 `aria-pressed="false|true"`，点击目标至少 `44 x 44 px`，保留现有可见焦点样式。

### 8.2 详情页

- 在详情页顶部现有“编辑 / 克隆 / 返回列表”操作组加入带文字的收藏按钮。
- 服务端渲染初始状态，并把 `isFavorite` 加入安全序列化的详情数据。
- 收藏行为复用列表页同一个本地脚本控制器和同一个接口，不在 `admin-detail.js` 建立第二套业务路径。

### 8.3 请求状态

- 点击后立即禁用当前按钮并设置忙碌状态，但不预先翻转星标。
- 只有服务端返回成功后才更新图标、文字、`aria-pressed` 与数据属性。
- 普通列表或详情页中原地更新，并显示“已收藏”或“已取消收藏”通知。
- “已收藏”筛选中取消收藏时，先显示成功通知，再刷新当前筛选 URL；服务端重新计算结果、总数与分页。
- 网络、鉴权、CSRF、校验或数据库失败时恢复按钮可用状态，保持原星标，并显示错误通知。
- 不添加持续动画、金黄色强调或内联可执行脚本。

## 9. 生命周期、导出与公开边界

- 创建：首页、Share API 与克隆创建的新分享都默认未收藏。
- 编辑：标题、正文、密码、过期时间和 Markdown 主题变化不改变收藏状态。
- 过期：不改变收藏状态，后台仍可查看和筛选。
- 删除：删除页面记录即删除其收藏状态，不需要额外清理表。
- 克隆：不继承收藏状态。
- 导出：每条记录增加布尔字段 `isFavorite`；导出仍覆盖全部分享，不因当前列表筛选而改变范围。
- 公开读取：`/view/:id`、公开元数据、浏览量事件与密码校验响应均不包含收藏状态。
- Share API：请求和响应保持兼容，不接受或返回收藏字段。

## 10. 失败处理

| 场景 | 行为 |
|---|---|
| Payload 不是布尔值 | `400`，不写数据库或审计 |
| Dashboard 会话无效 | `401` JSON，前端提示重新登录 |
| CSRF 无效 | `403`，状态不变 |
| 分享不存在 | `404`，状态不变 |
| 重复提交当前值 | `200`、`changed=false`，不写审计 |
| 收藏写入失败 | `500`，界面保持旧状态 |
| 收藏成功、审计失败 | 收藏保持成功；服务端记录审计错误 |
| 筛选中取消最后一项 | 刷新后显示组合筛选空状态；页码收敛到有效范围 |
| 过期分享收藏/取消收藏 | 与未过期分享相同，允许操作 |

- Memory 与 Postgres 均覆盖收藏、取消收藏、幂等提交、缺失 ID、列表投影和计数。
- 收藏条件分别与搜索、类型、保护状态和日期组合，列表数量与 `countPages()` 一致。

### 11.3 列表、详情与导出

- `favorite=true` 只返回收藏分享，并与其他条件组合。

- 克隆收藏分享后，新记录为未收藏。

## 13. 上线与回滚

上线顺序：

1. 对生产数据库做只读 schema 与页面数量快照，不读取分享正文。
2. 在可信迁移环境执行 `003`，确认历史行均为未收藏且业务行数不变。
3. 立即重复运行迁移，确认没有第二次变更。
4. 部署包含收藏功能的 Preview，执行后台鉴权、收藏、筛选、导出、克隆和公开边界 smoke。
5. 部署已提交的同一修订到 Production，并确认自定义域名指向该部署。
6. 在生产创建一条可删除的验证分享，完成收藏、筛选、取消收藏、审计和删除闭环。
7. 检查部署日志没有新增错误级事件，并清理验证数据。

回滚：

- 代码回滚到上一稳定部署；旧代码会忽略新增列。
- 保留 `is_favorite` 列和迁移记录，不做破坏性 schema 回退。
- 已有收藏数据保留，重新部署新版本后可继续使用。
- 若接口异常，优先回滚应用；不要删除页面或清空收藏列。

## 14. ADR 判断

本功能不创建 ADR：全局布尔收藏与当前共享后台身份模型一致，技术选择可通过兼容迁移扩展，也没有引入新的平台、数据库或跨上下文集成锁定。产品语义保存在 `CONTEXT.md`，实现与验证约束保存在本规格中即可。

## 15. 验收标准

功能只有在以下条件全部满足时才算完成：

- 收藏状态来自持久数据库，跨浏览器、跨实例和刷新保持一致。
- 列表与详情都能安全地收藏和取消收藏。
- “已收藏”可与所有现有筛选条件组合，结果、总数和分页一致。
- 取消收藏只在服务端成功后改变界面；收藏筛选结果会正确移除该行。
- 克隆不继承收藏，过期不清除收藏，删除自然移除收藏。
- 导出包含 `isFavorite`，公开页面与 Share API 不暴露收藏状态。
- 真实变化生成一次审计，幂等提交不生成审计；审计失败不回滚收藏。
- 第一版没有个人收藏、收藏时间、批量操作、统计卡片或公开精选列表。
- 全量测试、Postgres 集成测试、浏览器检查、Preview 和 Production smoke 全部通过。
